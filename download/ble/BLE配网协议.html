自定义AI生成 BLE 蓝牙配网协议

本文档定义 APP/小程序与硬件设备之间的蓝牙低功耗（BLE）配网交互协议。

一、配网流程概述

二、UUID 定义

用途 UUID 短格式 完整 UUID
服务 UUID 0xFA00 0000FA00-0000-1000-8000-00805F9B34FB
写特征 UUID (write without response) 0xEA02 0000EA02-0000-1000-8000-00805F9B34FB
读/通知特征 UUID (notify) 0xEA01 0000EA01-0000-1000-8000-00805F9B34FB

蓝牙技术联盟 SIG 基础 UUID：0000xxxx-0000-1000-8000-00805F9B34FB

三、数据协议格式

3.1 协议帧结构

字段 长度 说明
HEAD 1 byte 协议头，标识数据方向
TYPE 1 byte 消息类型
TOTAL 1 byte 总包数，从 1 开始
NUM 1 byte 当前分包序号，从 1 开始
LEN 1 byte 当前包有效数据长度 (0-58)
DATA 0-58 byte 数据内容（JSON 字符串）
CHK 1 byte 校验和

单包最大有效数据：58 字节

3.2 协议头 (HEAD)

值 方向 说明
0xA0 设备 → APP 设备发送给应用
0xB0 APP → 设备 应用发送给设备

3.3 校验和 (CHK)

校验和 = (0x100 - (除 CHK 外所有字节之和 & 0xFF)) & 0xFF

示例：

3.4 分包规则

  • 当数据长度 > 58 字节时，需拆分为多包发送
  • TOTAL 表示总包数
  • NUM 表示当前包序号（从 1 开始）
  • 接收方需按 NUM 顺序重组数据

四、APP/小程序 → 设备端

4.1 消息格式

DATA 部分为 JSON 字符串，结构如下：

字段 类型 必填 说明
type int 是 消息类型
value object 否 数据内容
ack int 否 应答信息（仅应答包使用）

4.2 消息类型定义

type 功能 value 说明
1 配网 WiFi 配置信息
2 开始扫描 无，请求设备扫描 WiFi
255 应答 ack 为收到的消息 type

4.3 配网消息 (type=1)

字段 类型 说明
encryption int 1=加密需密码，0=未加密不需密码
ssid string WiFi 名称，最大 32 字节
pwd string WiFi 密码，最大 64 字节
openId string 用户唯一标识，小程序 openId，最大 32 字节

4.4 开始扫描 (type=2)

连接 BLE 后设备可能主动扫描，APP 也可发送此消息请求重新扫描。

4.5 应答消息 (type=255)

字段 说明
ack 收到的消息 type 值

五、设备端 → APP/小程序

5.1 消息格式

与 APP→设备 相同，DATA 为 JSON 字符串。

5.2 消息类型定义

type 功能 value 说明
1 WiFi 列表 设备扫描到的 WiFi 列表
2 服务信息 服务器信息，配网过程使用
201 配网失败 无
202 配网成功 设备信息
255 应答 ack 为收到的消息 type

5.3 WiFi 列表 (type=1)

字段 类型 说明
ssid string WiFi 名称
rssi int 信号强度 (dBm)
encryption int 1=需密码，0=不需密码

5.4 服务信息 (type=2)

设备网络连接成功后发送给 APP。

5.5 配网失败 (type=201)

5.6 配网成功 (type=202)

字段 类型 说明
snCode string 设备唯一序列号
appVersion string 固件版本号
board string 主板型号
mac string 设备 MAC 地址

六、应答机制

  1. 接收方在完整接收数据包后，发送应答包
  2. 应答包 type=255，ack 为收到消息的 type 值
  3. 若检测到丢包（含超时未收齐），可在应答中携带需重传的包序号：
  1. 发送方超时未收到应答，应主动重传
